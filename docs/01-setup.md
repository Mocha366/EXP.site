# EXP. Web サイト 環境構築ガイド

このドキュメントでは、EXP. の公式 Web サイト（Next.js + React）をローカル環境で動かすための手順を説明します。

## 前提条件

以下のソフトウェアがインストールされていることを確認してください。

- Node.js 18 以上（LTS 推奨）
- npm または pnpm
- git

バージョン確認例:

```bash
node --version
npm --version
git --version
```

## リポジトリのクローン

作業したいディレクトリで以下を実行して、このプロジェクトを取得します。

```bash
git clone <このリポジトリのURL>
cd EXP.
cd site
```

> ルートに `tera/` ディレクトリもありますが、このドキュメントは `site/` 以下の Web サイトについて説明します。

## 依存関係のインストール

`site` ディレクトリで以下を実行します。

```bash
npm install
```

パッケージマネージャーとして pnpm / yarn を使いたい場合は、プロジェクト内の `package.json` を確認し、チームで統一してください。

## 開発サーバーの起動

```bash
npm run dev
```

起動後、ブラウザで `http://localhost:3000` にアクセスすると、EXP. のランディングページが表示されます。

## ビルド & プレビュー

本番と同じビルドをローカルで確認したい場合は以下を実行します。

```bash
npm run build
npm run start
```

デフォルトでは `http://localhost:3000` で本番相当の挙動を確認できます。

## よくあるトラブル

- 依存関係エラーが出る場合  
  → `node_modules` と `package-lock.json` を一度削除し、`npm install` をやり直してください。

- ポート 3000 がすでに使用されている場合  
  → `npm run dev -- -p 3001` のようにポートを変えて起動できます。

# 環境構築 RTA
このドキュメントでは、TERA の開発および実行に必要な環境構築手順について説明します。

## 前提条件
以下のソフトウェアがインストールされていることを確認してください。
- Node.js
- npm
- git

以下のコマンドを実行して、例のようにバージョンが返ってくればインストールされています。
```bash
node --version # v22.14.0
npm --version # 10.9.2
git --version # git version 2.49.0.windows.1
```

## リポジトリのクローン
GitHub からリポジトリをクローンします。以下のコマンドをコードを置きたいディレクトリで実行してください。
```bash
git clone https://github.com/isc-exp-2/tera.git
cd tera # クローンされたリポジトリのディレクトリに移動
```

## .env ファイルの設定
プロジェクトのルートディレクトリにある `.env.example` をコピーして `.env` ファイルを作成し、必要に応じて設定を変更してください。

ほかのプロジェクトメンバーに `.env` の中身を教えてもらうか、自分で Firebase コンソールから値を取得してください。


## 依存関係のインストール
プロジェクトのルートディレクトリで以下のコマンドを実行して、必要な依存関係をインストールします。
```bash
npm install
```

## 開発サーバーの起動
以下のコマンドを実行して、開発サーバーを起動します
```bash
npm run dev
```
起動メッセージに書かれている URL（通常は `http://localhost:3000`）にブラウザでアクセスして、アプリケーションが正しく動作していることを確認してください。
